\section{Summary}

This document presents comprehensive performance comparison tables for machine learning model classification of lightweight cryptographic (LWC) algorithms. The analysis covers four key metrics:

\begin{itemize}
    \item \textbf{F1-Score:} Harmonic mean of precision and recall, providing balanced measure of model performance
    \item \textbf{Precision:} Ratio of correctly predicted positive observations to total predicted positives
    \item \textbf{Recall:} Ratio of correctly predicted positive observations to all actual positives
    \item \textbf{Accuracy:} Overall correctness of the model across all classes
\end{itemize}

The algorithms tested include: AES, PRESENT, SIMON, XTEA, PRINCE, MSEA, LEA, RECTANGLE, and ASCONv2.

The machine learning models evaluated are: Decision Tree, Random Forest, SVM, KNN, and LightGBM.

File sizes range from 16 KB to 2048 KB, providing comprehensive analysis across different data volumes.

\section{Results and Conclusion}

\subsection{Key Findings}

\begin{enumerate}
    \item \textbf{Random Forest Superiority}: Random Forest emerged as the clear winner with the highest F1-score (93.6\%) and best consistency across all file sizes.
    
    \item \textbf{Tree-Based Models Dominance}: Tree-based models (Random Forest, Decision Tree, LightGBM) significantly outperformed distance-based (KNN) and margin-based (SVM) models.
    
    \item \textbf{Algorithm Variability}: Some algorithms (AES, ASCONv2) are easily distinguishable, while others (SIMON, PRINCE) present classification challenges.
    
    \item \textbf{File Size Impact}: Performance generally improves with larger file sizes, with optimal performance at 64KB-512KB range.
\end{enumerate}

\subsection{Why Random Forest is the Best Choice}

Random Forest demonstrates superior performance due to several key advantages:

\begin{itemize}
    \item \textbf{Ensemble Robustness}: Combines multiple decision trees to reduce overfitting and improve generalization
    \item \textbf{Feature Interaction Handling}: Effectively captures complex relationships between performance metrics
    \item \textbf{Outlier Resistance}: Less sensitive to noisy data points compared to single-tree models
    \item \textbf{Perfect Classification Rate}: Achieves perfect classification in 72.2\% of cases
    \item \textbf{Low Failure Rate}: Only 1.9\% of classifications result in poor performance
\end{itemize}

\subsection{Final Conclusion}

This comprehensive analysis of machine learning models for encryption algorithm classification conclusively demonstrates that \textbf{Random Forest is the optimal choice} for production deployment. With 93.6\% F1-score, 72.2\% perfect classification rate, and excellent consistency across all file sizes, Random Forest provides the most reliable solution for automated encryption algorithm identification.

The study reveals that performance metrics derived from execution characteristics are highly effective features for distinguishing between different cryptographic algorithms, and ensemble methods like Random Forest are best suited to capture the complex patterns in these metrics.

For organizations implementing automated encryption algorithm classification systems, Random Forest should be the primary model choice, with Decision Tree serving as a backup for scenarios requiring model interpretability.
